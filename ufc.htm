UFC Data Analytics

Data Analytics platform to source and display real time and historic fighter data.

Project

The project consisted of desigining a full end to end platform that would source large amounts of UFC fighter data, organize the data and display the data in a simple web application. The platform had to built in a way that would handle large data form the past and real time data of an ongoing fight. Aside from the data collection and view, the database had to be designed in a way to allow easy transformation and exportation for analysis.

System Architecture Overview

The system was broken down into subsystems that each handle a specific functionality for the application. A technique called subsystem decomposition was used to break down and identify the subsystem components that make up the whole system. This system decomposition only breaks down the system into basic functions. A second subsystem decomposition is performed to get a more detailed view of how the components will interact. The image below shows the overall architecture.

Flow Chart

The flow chart describes the usage and outcomes for each functionality on the user interface. This planning application is very complex and entails a large number of functions. A user flow chart was created for each page of the platform to allow for a smooth user experience.

Database Design

Database Design was the most critical part of the project and several designs and iterations were conducted before deciding on the final design. A complex database design could lead to scaling issues and introduce unnecessary hurdles, thus the design was built with simplicity in mind. Decomposing the subsystems decomposed into a UML class diagram to define interaction between provided a detailed insight into the control flow. Additionally, encapsulation was used to ensure security thorughout database design.

Final Design

The final system was built and deployed on AWS. Although no design is final due to constant changes in user needs, we can temporarily call this the final design. Hooray for the iterative design process!
